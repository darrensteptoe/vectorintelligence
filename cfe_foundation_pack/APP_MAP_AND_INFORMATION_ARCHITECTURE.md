# APP_MAP_AND_INFORMATION_ARCHITECTURE.md

## Design principle
CFE should feel like FPE's sibling: same family, same seriousness, same disciplined density.

The app should be organized around campaign workflow, not data type alone.

## Top-level navigation
1. Overview
2. Budget
3. Timeline
4. Benchmarks
5. Funding Path
6. Finance Operations
7. Donor Intelligence
8. Expenditure Intelligence
9. Risks
10. Reports
11. Manual
12. Settings / Data / Imports

## Page intent
### Overview
Leadership view. Must answer:
- What is the budget?
- What is funded?
- What is at risk?
- What needs action now?

### Budget
Where the operator defines the full campaign budget.

### Timeline
Where the operator sees when costs hit and where pressure builds.

### Benchmarks
Where the operator sees historical realism context.

### Funding Path
Where the operator sees raise targets, checkpoint requirements, and channel plan.

### Finance Operations
Where the operator manages call time, meetings, events, pledges, and tasks.

### Donor Intelligence
Where the operator studies support geography and occupation/industry mix.

### Expenditure Intelligence
Where the operator studies where money is going and whether it aligns with plan.

### Risks
Concentrated warning surface with actions.

### Reports
Generate and preview consultant-grade outputs.

### Manual
Persistent teaching layer for every major concept.

### Settings / Data / Imports
Bridge snapshots, source imports, classification tables, and export controls.

## Layout guidance
- Left rail: persistent nav and context switching.
- Center: primary work surface.
- Right rail: context, warnings, assumptions, interpretive notes, help, or current snapshot summary.
- No major module should depend on hidden content to be usable.
- The right rail should feel like FPE: compact, high-signal, always useful.

## Information hierarchy
At every page, the operator should know:
1. What page this is.
2. What question this page answers.
3. What state the campaign is in.
4. What matters most right now.
5. What action should happen next.

## Navigation rules
- No orphan pages.
- No deep hidden workflows for critical functions.
- Reports should be reachable from both the Reports page and relevant working pages.
- Manual/help should be reachable in context and from the main Manual page.
