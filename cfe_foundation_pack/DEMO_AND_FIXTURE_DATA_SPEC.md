# DEMO_AND_FIXTURE_DATA_SPEC.md

## Purpose
Provide realistic demo and fixture data so the product can be built, previewed, and tested meaningfully.

## Recommended demo campaigns
1. Low-cost local race
2. Legislative race with moderate budget complexity
3. Higher-cost race with significant consultant/media pressure

## Demo requirements
Each demo campaign should include:
- race setup
- budget lines
- spend timeline
- field bridge snapshot
- contribution samples
- expenditure samples
- ZIP patterns
- occupation strings
- vendor classification ambiguity
- warnings and risk states
- sample reports

## Fixture requirements
Fixtures should include:
- clean cases
- unknown cases
- ambiguous classification cases
- overbudget cases
- underfunded cases
- reserve stress cases
- bridge caution/redline cases
