# VISUAL_SYSTEM_AND_ECOSYSTEM_RULES.md

## Ecosystem principle
CFE should look and feel like FPE's sibling.
Not identical. Not reskinned. But unmistakably from the same ecosystem.

## Shared family traits
- restrained, professional visual tone
- dense but readable layouts
- strong card architecture
- disciplined spacing
- always-useful right rail
- businesslike typography
- clear warning hierarchy
- quiet confidence, not startup theatrics

## What to carry over from FPE
- same seriousness of information density
- same belief in high-signal cards
- same emphasis on canonical outputs
- same right-rail value model
- same full-background warning/status treatment
- same feeling that the product was built by operators, not by dashboard decorators

## What should differ
- CFE can lean slightly more ledger/report/table oriented.
- CFE should feel a little more executive-financial than operational-field.
- Donor and expenditure surfaces can be more tabular without losing ecosystem feel.

## Visual rules
- Avoid gimmicky widgets.
- Avoid oversaturated color use.
- Use status color to convey meaning, not decoration.
- Dense tables must remain legible.
- Report preview surfaces should feel polished and circulation-ready.
- Every major page should have a clear page intro and a clear “what this page is for” explanation.

## Warning philosophy
- Good = calm confidence
- Caution = meaningful friction
- Bad = clear operational concern
- Unknown = visible uncertainty, not hidden uncertainty

## Manual styling philosophy
The manual/help layer should feel like part of the product, not bolted on.
It should be:
- dense
- practical
- human
- campaign-literate
- interpretive
- calm
